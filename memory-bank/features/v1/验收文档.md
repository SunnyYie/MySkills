# 使用手册

## v1 可用功能

当前 v1 可以当作一个本地 CLI 编排与记录工具来使用。

可直接使用的能力：

- `bind project|jira|requirements|gitlab|feishu|repo`
  - 维护项目画像配置
- `inspect config|graph|connectors`
  - 检查配置完整性、关系图和 connector 状态
- `run start`
  - 创建一个 bugfix run，并初始化 checkpoint
- `run brief`
  - 创建一个 `brief_only` 子工作流 run
- `run status`
  - 查看当前 run 状态、等待原因、可执行动作
- `run resume`
  - 从最近 checkpoint 恢复
- `run approve|reject|revise`
  - 记录审批、拒绝、回退
- `run provide-artifact`
  - 补录 GitLab 产物
- `run provide-verification`
  - 补录验证结果
- `run preview-write`
  - 生成本地写回 preview artifact
- `run execute-write`
  - 生成本地写回 result artifact
- `record jira`
  - 创建仅 Jira 回写子工作流 run
- `record feishu`
  - 创建仅飞书记录子工作流 run

## v1 最简操作方式

1. 先绑定项目配置

```bash
bugfix-orchestrator bind project --project <project_id> --file <project.json>
bugfix-orchestrator bind jira --project <project_id> --file <jira.json>
bugfix-orchestrator bind requirements --project <project_id> --file <requirements.json>
bugfix-orchestrator bind gitlab --project <project_id> --file <gitlab.json>
bugfix-orchestrator bind feishu --project <project_id> --file <feishu.json>
bugfix-orchestrator bind repo --project <project_id> --file <repo.json>
```

2. 检查配置

```bash
bugfix-orchestrator inspect config --project <project_id>
bugfix-orchestrator inspect graph --project <project_id>
bugfix-orchestrator inspect connectors --project <project_id>
```

3. 启动 run

```bash
bugfix-orchestrator run start --project <project_id> --issue <JIRA-123>
```

4. 查看 run 状态或恢复

```bash
bugfix-orchestrator run status --run <run_id>
bugfix-orchestrator run resume --run <run_id>
```

5. 人工补录产物或验证结果

```bash
bugfix-orchestrator run provide-artifact --run <run_id> --file <artifact.json>
bugfix-orchestrator run provide-verification --run <run_id> --file <verification.json>
```

6. 预览或执行本地写回流程

```bash
bugfix-orchestrator run preview-write --run <run_id> --stage "Artifact Linking"
bugfix-orchestrator run execute-write --run <run_id> --stage "Artifact Linking" --preview-ref <preview_ref>
```

## v1 使用边界

v1 现在适合用于：

- 维护项目配置
- 创建和管理 bugfix run
- 做审批、回退、恢复
- 补录 branch / commit / MR / verification 信息
- 做本地 preview、artifact 落盘和流程演练

v1 现在还不适合直接当成真实生产集成来用：

- `run start` 还不会自动走完整主流程
- 还没有真实 Jira 读取接到主流程
- `preview-write` / `execute-write` 目前是本地 artifact，不是真实 Jira / 飞书写入
- 还没有完整的 `Requirement Brief` 和 `Bugfix Report` 对外交付命令
