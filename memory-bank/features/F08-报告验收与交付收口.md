# F08 报告验收与交付收口

## 1. 目标与用户价值

本 feature 负责把整个 v1 的实现、验证和交付收束为统一的报告、追踪矩阵、验收清单和交付包。

用户价值如下：

- 用户可以通过一份 `BugfixReport` 回看一次 run 的完整结论。
- 开发人员和验收人员可以清楚知道每条需求对应什么验证路径。
- 交付物和成功指标具备明确统计口径，而不是停留在口头说明。

## 2. 来源依据

### 2.1 已确认事实

- `BugfixReport` 是 run 的统一最终输出。
- 需求文档中的 v1 验收场景必须全部有对应实现落点和验证路径。
- 成功指标必须可采样、可计算、可复核。

### 2.2 上游约束

- 技术方案把 Markdown / JSON 输出格式化交给 Renderers。
- 实施计划任务八冻结了 `BugfixReport` 最小内容、追踪矩阵、测试套件结构、成功指标、最终验收清单和交付包清单。
- 当前仓库仍以文档先行为主，因此收口文档必须能服务后续代码实现与验收。

### 2.3 本 Feature 实现决策

- `BugfixReport` 作为最终人工可读与可导出的统一出口。
- 追踪矩阵、测试结构、验收清单和交付包清单分别回答不同问题，不互相替代。
- 收口文档不重新定义业务流程，只复用 F04-F07 的事实结果。

## 3. 范围

### 3.1 本 Feature 要做什么

- 定义 `BugfixReport` 的最小内容与输出口径。
- 定义需求到测试追踪矩阵。
- 定义测试套件层次。
- 定义成功指标与证据要求。
- 定义最终验收清单与交付包清单。

### 3.2 本 Feature 明确不做什么

- 不重新设计主流程阶段。
- 不承担 preview / execute 细节。
- 不替代 CLI 命令入口。
- 不放宽 v1 非目标。

## 4. 前置依赖与输出产物

前置依赖：

- F04 提供报告内容主体
- F05 提供状态与审批历史
- F06 提供写回结果
- F07 提供标准演示路径与 CLI 验收入口

输出产物：

- `BugfixReport`
- 需求到测试追踪矩阵
- 分层测试结构
- 成功指标说明
- 最终验收清单
- 交付包清单

## 5. 涉及的核心对象、状态与能力边界

核心对象：

- `BugfixReport`
- `ExecutionContext`
- `ApprovalRecord`
- `StructuredError`

关键字段：

- 最终状态
- 审批历史
- 开放风险
- Jira 回写摘要
- 飞书记录摘要

能力边界：

- report-writer 汇总结果，但不重新计算 workflow 状态真相。
- 验收矩阵负责把需求映射到测试，不替代测试执行本身。
- 成功指标依赖运行与审计证据，不依赖人工回忆。

## 6. 实现设计

### 6.1 主要流程

主要流程如下：

1. 从分析产物、写回结果、审批记录和最终状态汇总 `BugfixReport`。
2. 根据需求文档与实施计划建立追踪矩阵。
3. 将测试划分为领域模型、持久化与安全、skill、workflow、connector 契约、CLI 集成、需求追踪与指标验证、验收场景。
4. 基于标准演示路径和关键场景建立最终验收清单。
5. 汇总交付包清单，保证交付说明与实现结果一致。

### 6.2 输入与输出

输入：

- `RequirementBrief`
- 代码定位与修复计划
- verification 结果
- Jira / 飞书写回摘要
- 审批历史
- 最终状态

输出：

- `BugfixReport`
- traceability matrix
- acceptance checklist
- delivery package checklist
- success metrics definition

### 6.3 关键约束

- 报告必须足以支撑复盘、验收和知识沉淀。
- 每条需求都必须有至少一条验证路径。
- 测试结构要体现单元、契约、集成和验收分层。
- 成功指标必须有统计口径、样本范围和阈值门禁。

### 6.4 异常与失败分支

- 报告内容缺失：阻止声明“已完成交付”。
- 验收清单缺少关键场景：视为验收口径不完整。
- 指标无法计算：视为交付证据不足。
- 追踪矩阵缺失映射：视为需求与验证脱节。

## 7. 开发任务拆解

1. 定义 `BugfixReport` 最小字段与输出格式。
2. 建立需求到测试追踪矩阵模板。
3. 定义测试套件分层及其职责。
4. 定义成功指标及证据来源。
5. 定义最终验收清单。
6. 定义交付包清单并与实际产物对齐。

## 8. 测试与验收

单元/契约测试：

- `BugfixReport` 最小内容
- traceability matrix 字段完整性
- success metrics 字段完整性
- acceptance checklist 与 delivery package checklist 完整性

集成测试：

- 标准演示路径可产出最终报告
- v1 关键场景都能映射到验证动作与证据产物

文档一致性检查：

- 不引入超出 v1 的新交付能力
- 与 F04、F05、F06、F07 的事实来源一致

验收标准：

- `BugfixReport` 可作为 run 的统一最终输出
- 需求、测试、证据和交付物之间形成完整闭环

- 从 F05 接收最终状态与审批历史。
- 从 F06 接收写回结果与恢复证据。
- 从 F07 接收 CLI 验收入口和标准演示路径。
